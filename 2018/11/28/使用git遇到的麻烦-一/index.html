使用git遇到的麻烦1

参加一个学长的微信小程序项目，第一次团队协作的我面对git有些无助，踩了很多坑，记录下来。

基本知识

git pull用于在你今天的编码前，能够将现有的项目‘拉’下来，这样就获得了项目的最新代码。

git push用于提交代码到远端

问题:
遇到这个问题一定不要直接用git push -f,这个指令能不用就千万别用。这时候最好的办法就是直接clone项目到一个新文件夹，然后把你这次修改添加删除的代码改到新文件夹中，然后add->commit->push,这样就可以成功Push.我之前遇到这个问题时，直接再pull项目一边然后就…..整个项目直接冲突，到处都===<<<这些符号,所以最好的办法就是clone->commit->push

总结:近期用到的git指令

  1. git pull origin master–获取项目分支(master)
  2. git push origin master–将本地代码推到项目分支中
  3. git log –查看记录
  4. git reset –head<版本号>—救命！
  5. git branch –查看分支
  6. git status –查看状态
  7. git remote –查看远程库
  8. git remote add origin <ssh/http>–添加远程仓库
