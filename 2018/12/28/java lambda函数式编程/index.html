java lambda函数式编程

函数式编程一直都吸引着我,java8中引入了stream以及lambda表达式让我们能够用java写出流利的函数式代码,现在把主要的用法记录一下

在java8中Collection这个接口中，新增了一个default方法stream(),使得Collection下的所有数据结构实现的实例都可以转化为’stream’流,
通过操作stream提供的接口我们可以完成相应的函数式编程.

  1. 将一个ArrayList转化为stream,

接下来分析常用的api

  1. stream.map() 用于将流中的元素映射为该元素的属性，例如nameList.stream().map(name->name.length)中，将name映射为name的长度,映射结束后，流就变成各个name长度的流
  2. stream.filter() 用于将过滤流中的元素,例如nameList.stream().filter(name->name.length>=5),上述代码用于过滤流，使得剩下的流中的元素(name)的长度大于等于5
  3. stream.collect() 用于switch，使其转化为实际可用的数据结构,例如.collect(Collectors.toList())将流转化为List
  4. stream.sorted() 用于自定义流的顺序
  5. stream.distinct() 用于去除重复的流
  6. stream.findAny() 读名称就可以知道该方法用于查找指定的流，这里需要注意的是返回的是Optional的值，Optional也是java8中新出现的一种数据结构，可以防止空指针报错，使用myOptional.isPresent()返回true,则非空，使用myOptional.get()就可以获取到相应的值，返回false，则说明是空的
  7. stream.flatMap() 该方法和map()很像，但不同的是，flatMap()可以分解map后集合结构或者数组,这样说有点抽象，举个例子,定义一个family类，其中有一个属性为LinkedList children_names,该属性是一个保存家庭中所有孩子姓名的LinkedList.现在，我们有一个包含很多家庭的Array叫familiesArray,使用Stream.of(familiesArray).flatMap(family->family.getChildren_names)返回的流的基本单位不是LinkedList,而是String(name)!,flatMap()将map到的children_names flat到 单个的name,于是结果返回的stream是包含所有家庭所有孩子姓名的流

常见的终止操作

我们操作流来实现函数式编程的最终目的就是将其转化为有用的数据结构，所以这就需要各种终止操作,常见的终止操作有以下几种

  1. forEach()
  2. reduce()，这是一个用于累计的方法
  3. collect,上面已经提到了，用于将流转化为各种数据结构
  4. max(),count()……类似的方法还有很多

说了这么多的,labmda到底是什么，为什么使用->就可以实现函数式编程?

其实归根结底，labmda表达式的出现就是为了解决java编程中参数为方法的情况,例如如果不使用lambda，我们要自定义排序，我们需要在.sort()方法中new actualSort(),lambda表达式就是一个匿名方法，提供参数，然后执行对于的操作.
