blade应用部署之路

完成了一个工具类的web应用,要想使用就必须将其部署到我的服务器上面。今天踩了很多坑，但总算是完完整整疏通了一遍web应用的部署过程，如今总算可以说自己的”全栈工程师”了,hhh

部署思路->使用maven打包为jar包，然后将jar包发送到linux服务器，运行jar包

  1. 修改application.properties。 主要是加上server.address='主机公网ip'，以及修改mysql数据库密码
  2. 在pom.xml中添加”build”模块,需要注意的是一定要注意Application class的位置，比如我这次的项目中它是放到com.bladejava.Application中的
  3. 切换到项目目录，运行mvn clean package -DskipTestsjar包位于target文件夹中
  4. jar包也有了，这个时候就需要上传到服务器运行它。
  5. 这个过程中也用到了很多的linux指令,
    1. 例如mkdir [文件名]->创建文件夹,
    2. cp [旧文件] [新文件地址]->将旧文件复制到新文件地址,
    3. unzip [文件名]->解压zip文件至当前地址.
    4. 运行jar包，java -jar filename.jar.
    5. 查看进程 ps a, 杀死进程 kill -9 [pid]
    6. 不挂断运行程序 sudo nohup java -jar filename.jar & (注意加上sudo提供权限，后面一定要加上&)
  6. 需要注意的是maven打包项目的时候会自动使用本地环境变量中的jdk版本，所以如果项目jdk和本地jdk不一样的话就需要手动更改

思考与总结

部署项目时及时用git保存分支，这样就算项目打包失败也可以补救。

使用git遇到的麻烦1

参加一个学长的微信小程序项目，第一次团队协作的我面对git有些无助，踩了很多坑，记录下来。

基本知识

git pull用于在你今天的编码前，能够将现有的项目‘拉’下来，这样就获得了项目的最新代码。

git push用于提交代码到远端

问题:
遇到这个问题一定不要直接用git push -f,这个指令能不用就千万别用。这时候最好的办法就是直接clone项目到一个新文件夹，然后把你这次修改添加删除的代码改到新文件夹中，然后add->commit->push,这样就可以成功Push.我之前遇到这个问题时，直接再pull项目一边然后就…..整个项目直接冲突，到处都===<<<这些符号,所以最好的办法就是clone->commit->push

总结:近期用到的git指令

  1. git pull origin master–获取项目分支(master)
  2. git push origin master–将本地代码推到项目分支中
  3. git log –查看记录
  4. git reset –head<版本号>—救命！
  5. git branch –查看分支
  6. git status –查看状态
  7. git remote –查看远程库
  8. git remote add origin <ssh/http>–添加远程仓库

记录一下使用'blade'以及其配套orm库'anima'连接数据库的踩坑之旅

blade是国产的开源java web框架,github地址。

blade非常轻量,内置服务器，而且没有使用j2ee那一套,blade吸收了很多web框架的优点，开发速度非常快。blade的’hello world’

好了，优点说了这么多，但实际用起来也经历了不少坑，在博客里记录一下把.

目标：搭建一个blade项目，并连接好mysql数据库

  1. blade必须使用maven来创建项目,我用的idea，点击左上角File->new Project->选择maven->一直next就行了
  2. 关于maven：maven就相当于是java的包管理器，类似于python的pip,nodejs的npm
  3. 创建成功后，我们就可以导入项目所需的相关依赖了，核心的包括blade，mysql，anima等包,当时因为怕没有把全部依赖包导入进项目，我直接用了一个完整的blade项目的全部依赖,不怕多，就怕少。
  4. 这里需要注意的是我们应该使用最新的blade版本，我目前使用的是
    <blade.version>2.0.11.ALPHA</blade.version>这个版本,新版本启动项目用的是blade.of(),老版本用的是blade.me(),所以一定要用新版本
  5. 项目创建成功后，我们就该创建完整的目录结构了。。
  6. 目录结构分析一:com.bladejava是存放所有项目代码的包,在这个包下创建的Application.java文件就是启动文件。在com.bladejava下创建config,controller,model这几个包，config是用于配置的包，里面的Java文件用于项目配置(例如数据库的配置),我只有一个StartUpConfig.java是启动项目时需要运行的文件，通常还应该有处理异常的文件。controller存放路由文件,相当于flask中的蓝图。model存放模型。
  7. 目录结构分析二:和java文件夹同级有个resource,里面是存放静态资源。同样的新建两个目录,static,templates。static放静态资源(js,css)，templates放html模板。
  8. 目录结构分析三:resources下新建一个文件application.properties,这个就是配置文件了，里面存放各种配置信息，如mysql的配置，端口的配置等

项目目录搭建完成后，我们可以试一试了，再controller中新建一个indexController.java @Path是blade定义的一个注解，标明这是一个路由路径,@Path后面可以带参数。@GetRoute定义了一个路由。

使用hexo的注意事项

hexo博客确实好用，使用github托管，免去了云主机的费用，同时操作起来也方便。但最近也出现了一些问题，在这里记录下来。

一:修改网站信息

想要将博客网站中间的hexo四个大字换掉，就需要打开config.yml文件，修改前几行的信息就可以了

二:网站css和js加载不出来

这个问题最近一直缠着我，我当前的解决方法是修改config_yml文件的url板块,css和js加载不出来的原因是src属性后面需要的链接地址和实际的不符，常见的错误就是多了“你的用户名/”所以，可以试试将config_yml文件的root:那一行只留下root: /

三:加载图片的问题

我们在markdown文件中使用了本地图片，要把它加载到博客中就需要自己把图片放到source文件夹中，我的做法是在source中新建一个pictures文件夹,把图片放进去。同时markdown中图片应该这样写(../pictures/pic_name.jpg)
